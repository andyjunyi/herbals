## 1. 專案結構說明

```
HERBALS/
├── index.html        網站主頁面（HTML 結構）
├── styles.css        網站樣式（字體、顏色、版面）
├── app.js            網站邏輯（搜尋、篩選、顯示）
├── data.js           資料庫（所有方劑、藥材、茶飲資料）
│
├── update.js         自動更新腳本
├── update.bat        Windows 快捷方式（雙擊即可執行）
├── package.json      Node.js 設定
│
├── *.md              中藥研究參考資料（不直接顯示在網站）
└── .gitignore        Git 忽略清單
```

### 各檔案職責

| 檔案 | 什麼時候需要修改 |
|------|-----------------|
| `data.js` | 新增／修改方劑、藥材、茶飲資料 |
| `styles.css` | 調整顏色、字體、間距、排版 |
| `app.js` | 變更搜尋邏輯、顯示方式、功能 |
| `index.html` | 修改頁面結構、標題、標籤 |

## 2. 日常更新流程

### 情境 A：新增一篇中藥研究 .md 檔案（有 API 金鑰）

```
1. 將新的 .md 檔案放入 HERBALS 資料夾
2. 雙擊 update.bat
3. 腳本自動解析並推送至 GitHub，完成
```

### 情境 B：手動新增一筆資料（無需 API 金鑰）

```
1. 雙擊 update.bat
   （或在終端機輸入：node update.js --add）
2. 依照問答畫面填入資料
3. 填完後自動推送至 GitHub，完成
```

### 情境 C：只修改了網站檔案（styles.css / app.js 等）

```
1. 完成修改後，開啟終端機
2. cd "d:/My Projects/HERBALS"
3. node update.js --push-only
```

## 3. 使用自動更新腳本

### 前置設定（只需做一次）

**步驟 1：取得 Anthropic API 金鑰**
1. 前往 https://console.anthropic.com
2. 登入 → 點選左側「API Keys」
3. 點「Create Key」，複製金鑰（格式：`sk-ant-...`）

**步驟 2：設定環境變數**  
開啟「命令提示字元（cmd）」（非 PowerShell），執行：
```
setx ANTHROPIC_API_KEY "sk-ant-你的金鑰"
```
關閉後重新開啟終端機，設定生效。

> ⚠️ 金鑰絕對不能寫入程式碼或上傳至 GitHub（詳見第 8 節）

### 腳本指令一覽

| 指令 | 說明 |
|------|------|
| `node update.js` | 掃描新 .md 檔，解析後推送 |
| `node update.js --add` | 手動填寫新增資料，推送 |
| `node update.js --preview` | 預覽將新增什麼，不寫入 |
| `node update.js --no-push` | 更新本地資料，不推送 GitHub |
| `node update.js --push-only` | 不解析 .md，直接推送目前變更 |

### 費用說明

腳本使用 Claude Haiku 模型（最便宜）：
- 每解析一個 .md 檔案：約 **0.003 美元**（新台幣不到 0.1 元）
- 每個檔案只解析一次，修改後才會重新解析
- 新帳號有 **5 美元免費額度**

## 4. 手動新增資料（不需 API）

執行 `node update.js --add` 後，依照提示填寫：

### 新增藥材範例

```
請選擇資料類型：
  1. 藥材（herb）
  2. 養生茶飲（tea）
  3. 方劑（formula）

輸入 1 / 2 / 3：1

藥材名稱：薑黃
分類（如：補氣藥、清熱藥）：活血化瘀藥
藥性（溫/平/涼/寒）：溫
味道（如：甘、苦）：辛、苦
歸經（如：脾、肺）：脾、肝
功效（如：補氣健脾、止咳）：活血行氣、痛經止痛
主治：血瘀氣滯所致的胸脅刺痛、月經不調
禁忌：陰虛內熱者慎用
常見搭配（可留空）：當歸、川芎

── 適用症狀 ──
  1.消化不良  2.便秘腹脹  3.失眠多夢  ...
輸入編號：6,11
```

### 填寫技巧

- **多個項目**用「、」或「,」分隔（如：`脾、肝` 或 `脾,肝`）
- **可留空的欄位**直接按 Enter 跳過
- **症狀**輸入數字編號（如：`1,6` 代表消化不良＋氣虛疲勞）
- **季節**輸入 1春 2夏 3秋 4冬，留空代表全年適用

## 6. 修改網站外觀

### 修改主題色

開啟 `styles.css`，找到最上方的 CSS 變數區塊：

```css
:root {
  --primary:      #8B2000;   /* 主色（深紅） */
  --accent:       #C9860A;   /* 強調色（金黃） */
  --tea-green:    #4a7c59;   /* 茶飲專用綠色 */
  --bg:           #fdf8f0;   /* 頁面背景色 */
}
```

### 修改字體大小

`styles.css` 最上方：
```css
html {
  font-size: 112.5%;  /* 調整這個數值（100% = 16px，112.5% = 18px）*/
}
```

### 修改後同步

```bash
node update.js --push-only
```

## 7. 同步至 GitHub

### 方法 A：使用腳本（推薦）

```bash
node update.js --push-only
```

### 方法 B：手動 Git 指令

```bash
cd "d:/My Projects/HERBALS"

# 查看有哪些檔案變更
git status

# 加入所有變更
git add -A

# 提交（引號內填寫說明）
git commit -m "說明這次修改的內容"

# 推送至 GitHub
git push
```

### GitHub Pages 部署

網站推送後，GitHub Pages 約 1～2 分鐘自動更新。  
若尚未啟用 GitHub Pages：

1. 進入 https://github.com/andyjunyi/herbals
2. 點選上方 **Settings**
3. 左側選單 → **Pages**
4. Branch 選 `main`，資料夾選 `/ (root)` → **Save**
5. 網站網址：`https://andyjunyi.github.io/herbals/`

## 8. 安全注意事項

### ⚠️ API 金鑰安全守則

| 禁止事項 | 原因 |
|----------|------|
| 將金鑰寫入程式碼 | 推送 GitHub 後全球可見，他人可盜用 |
| 將金鑰存入任何 .js / .html 檔案 | 同上 |
| 將金鑰透過訊息傳送給他人 | 可能被截取 |
| 在公共場所輸入金鑰 | 可能被側錄 |

### ✅ 正確做法

金鑰只存放在**系統環境變數**：
```
setx ANTHROPIC_API_KEY "sk-ant-你的金鑰"
```
程式透過 `process.env.ANTHROPIC_API_KEY` 讀取，金鑰不出現在任何程式碼中。

### 發現金鑰外洩時

1. 立刻前往 https://console.anthropic.com/settings/keys
2. 找到該金鑰，點 **Revoke（撤銷）**
3. 重新建立新金鑰並重新設定環境變數
4. 若已推送至 GitHub，聯繫 GitHub 說明（可申請清除 git history）

### GitHub 存取金鑰（個人 Token）

若 `git push` 需要密碼驗證，建議設定 **Personal Access Token**：
1. https://github.com/settings/tokens → Generate new token
2. 勾選 `repo` 權限 → 複製 Token
3. Git 要求密碼時，貼上 Token（不是 GitHub 密碼）

## 9. 常見問題排解

### Q：執行 `node update.js` 出現「未設定 ANTHROPIC_API_KEY」

**解法：**
1. 確認已在 cmd 執行 `setx ANTHROPIC_API_KEY "sk-ant-..."`
2. **關閉並重新開啟**終端機（setx 須重啟才生效）
3. 執行 `echo %ANTHROPIC_API_KEY%` 確認有輸出值

---

### Q：`git push` 要求輸入帳號密碼

**解法：** 使用 Personal Access Token（見第 8 節）或設定 Git credential：
```bash
git config --global credential.helper store
```
下次輸入一次後即記住。

---

### Q：網站資料沒有更新（推送成功但網站看起來一樣）

**原因：** 瀏覽器快取舊資料  
**解法：** 按 `Ctrl + Shift + R`（強制重新載入），或開無痕視窗確認

---

### Q：data.js 編輯後網站顯示空白或出錯

**原因：** JavaScript 語法錯誤  
**解法：**
1. 用瀏覽器開啟網站，按 `F12` → Console 查看錯誤訊息
2. 常見問題：忘記逗號、引號未成對、中文括號

---

### Q：新增的資料沒有出現在網站

**可能原因 1：** 瀏覽器快取了舊的 localStorage 資料  
**解法：** 點網站右上角的「↺ 重置」按鈕，重新載入初始資料

**可能原因 2：** GitHub Pages 尚未更新  
**解法：** 等待 1～2 分鐘後再重新整理

---

### Q：想要刪除某筆資料

直接在 `data.js` 找到該筆資料，刪除整個 `{ ... }` 區塊（含末尾逗號），  
然後執行：
```bash
node update.js --push-only
```
網站上需另外點「↺ 重置」讓 localStorage 更新。
